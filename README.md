# CS 320 Portfolio

## Reflection

Throughout this course, I learned that testing is one of the most important parts of developing software. Writing JUnit tests helped me make sure my code worked the way it was supposed to and that it handled invalid inputs correctly. I also learned that validating user input, following the project requirements, and testing each feature as I built it helped create software that was both functional and more secure. Finding problems early made them much easier to fix than waiting until the end of the project.

Interpreting user needs starts with understanding the requirements before writing any code. For each project, I looked at what the program needed to do, broke the requirements into smaller tasks, and built one feature at a time. I used those requirements to guide my testing so I could confirm everything worked as expected. This made it easier to stay organized and make sure I wasn't missing anything.

When I design software, I like to plan out the structure before I start coding. Keeping classes organized and giving each one a specific purpose made the projects easier to build, test, and update later. This course showed me how valuable unit testing and automation are for creating reliable software, and I'll continue using those practices in future projects.
